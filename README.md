## 📋 项目简介

本 Skill 是专为政府数字化项目设计的售前顾问助手，基于 OpenClaw / QClaw 框架运行。它覆盖从机会发现、方案设计、投标执行到交付交接的完整流程，帮助售前人员快速响应政府项目需求，提升方案质量和投标成功率。

### 核心能力

- ✅ **机会评估**：基于市场吸引力、竞争态势、执行能力三维评估，提供 Go/No-Go 决策支持
- ✅ **方案设计**：提供标准化的技术方案大纲，自动包含等保、密评、信创等合规章节
- ✅ **投标支持**：标前检查清单，识别废标风险，确保投标文件完整性
- ✅ **合规设计**：等保三级、密评、信创适配、数据安全等政务项目必备合规项
- ✅ **POC 支持**：POC 方案设计指南，包含演示脚本和验收标准

## 🎯 适用场景

- 智慧城市、数字政府、一网通办、城市大脑项目
- 政务云、政务数据平台、公共数据开放项目
- 信创适配、等保合规、密评相关项目
- 政府项目投标、售前方案、标书制作
- 政府项目 POC 验证

## 🚀 安装方法

### 前置要求

- OpenClaw v0.3.0+ 或兼容 Skill 加载机制的 Agent 框架
- 支持 Skill 机制的运行环境

### 安装方式

#### 方式一：npm 安装（推荐）

```bash
npm install @prince90s/gov-digital-presales
```

安装完成后，Skill 会自动部署到 OpenClaw 的 skills 目录中。

#### 方式二：手动安装

1. **克隆或下载本项目**

```bash
git clone https://github.com/prince90s/gov-digital-presales.git
```

2. **部署到 OpenClaw**

将 `gov-digital-presales` 文件夹完整放入 OpenClaw 的 `skills/` 目录下：

```bash
cp -r gov-digital-presales ~/.openclaw/skills/
```

3. **重启/重载会话**

在 OpenClaw 中执行会话重启或重载 Skill 命令，框架会自动加载该 Skill。

4. **验证加载状态**

发送 `查看当前已加载的 Skill`，确认 `gov-digital-presales` 已在列表中显示。

## 🔄 工作流程

本 Skill 采用 5 步标准化工作流程：

### Step 1：机会评估（Go/No-Go）

1. 提取项目基本信息（名称/预算/截止时间/采购单位）
2. 按三个维度打分：市场吸引力(40%) + 竞争态势(30%) + 执行能力(30%)
3. 总分 ≥80 → 建议参与；60-79 → 有条件参与；<60 → 不建议参与
4. 输出评估结论和短板弥补建议
5. **请用户确认**："是否继续推进该项目？"

### Step 2：需求调研与方案设计

1. 询问客户层级（省级/地市/区县）、业务方向、预算规模
2. 按技术方案大纲组织方案结构
3. 方案必须包含独立章节：等保三级、密评、信创适配
4. 输出前用标前检查清单自检
5. **请用户确认**："方案框架是否满意？"

### Step 3：投标支持

1. 分析标书评分标准，识别高分项（通常技术方案40-50%，资质10-15%）
2. 逐条响应技术参数表（"完全响应"/"部分响应"/"不响应"明确标注）
3. 用标前检查清单做最终检查（两人交叉验证）
4. 提醒关键废标风险（保证金/社保证明/信用报告时限）
5. **请用户确认**："标书内容是否完整？"

### Step 4：合规设计（等保/密评/信创）

1. **等保三级**：安全架构图 + 5个安全类逐条响应 + 日志留存≥6个月 + RTO/RPO量化
2. **密评**：独立章节 + SM2/SM3/SM4 使用场景清单 + 国密产品认证号
3. **信创**：分阶段替代计划 + 信创目录内产品选型 + 兼容性测试报告
4. **数据安全**：数据分类分级方案 + 个人信息保护设计

### Step 5：POC 验证支持

1. 设计 POC 范围确认表（必须客户签字，防止范围蔓延）
2. 输出演示脚本（分决策层/业务层/技术层三个版本）
3. 制定验收标准（量化：功能点/接口成功率/响应时间/数据准确性）
4. POC 完成后输出总结报告（可作后续投标素材）
5. **请用户确认**："POC范围确认表内容是否准确？"

## ⚠️ 使用边界

### ✅ 适用场景

- 政务数字化项目售前阶段的方案编写
- 投标响应、合规梳理、POC设计
- 智慧城市、政务云、大数据平台等政务类项目

### ❌ 不适用场景

- 已中标项目的开发排期
- 非政务行业（如互联网ToC）项目方案
- 涉密等级高于公开可参考级别的项目

### ⚠️ 重要提示

本 Skill 提供的模板与内容为通用政务项目参考框架，实际投标/申报需结合项目招标文件与当地主管部门要求调整，最终内容需由专业售前/法务人员审核确认。
