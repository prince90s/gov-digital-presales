# POC 方案设计指南

---

## 一、POC 战略意义

### 何时主动提出 POC？
- 客户对核心技术存在疑虑（"能对接我们的老系统吗？"）
- 竞争对手方案与我方高度相似
- 项目金额较大（>500万），客户决策谨慎

### POC 核心目标（按优先级）
1. 验证技术可行性（一票否决项）
2. 展示差异化价值（加分项）
3. 建立客户信心（信任项）
4. 锁定技术方案（商务项）

---

## 二、POC 范围界定（关键！）

> ⚠️ POC范围必须书面确认，避免范围蔓延。

**POC范围确认表**（需客户签字）

| 项目 | 内容 |
|------|------|
| POC目标 | 验证与现有系统的对接可行性 |
| 功能范围 | 1.单点登录 2.数据同步 3.状态回传 4.统一认证 |
| 非功能要求 | 并发≥100用户，响应≤3秒，7×24可用 |
| 数据范围 | 使用脱敏数据，约5000条 |
| 环境要求 | 我方提供云环境；客户远程访问 |
| POC周期 | 自环境就绪起15个工作日 |
| 验收标准 | 见第三节 |
| 双方责任人 | 我方：XXX；客户方：XXX |

## 三、POC 验收标准（必须量化）

| 验收项 | 验收标准 | 验收方式 | 通过 |
|--------|---------|---------|------|
| 功能完整性 | X个功能点全部可演示 | 现场演示+功能清单 | □通过 |
| 接口联通性 | 接口联通成功率≥99% | Postman测试报告 | □通过 |
| 性能达标 | 100并发下平均响应≤3秒 | JMeter压测报告 | □通过 |
| 数据准确性 | 数据同步准确率=100%（误差≤0.1%） | 数据比对报告 | □通过 |
| 安全合规 | 通过等保二级测评 | 安全测评报告 | □通过 |

## 四、演示脚本设计

### 4.1 标准演示流程（45分钟）

| 时间段 | 演示内容 | 讲解要点 | 负责人 |
|--------|---------|---------|--------|
| 0-5分钟 | POC环境介绍+架构说明 | 安全隔离、数据脱敏 | 售前工程师 |
| 5-20分钟 | 核心业务流程演示 | 用真实脱敏数据 | 售前工程师 |
| 20-30分钟 | 技术亮点展示 | 性能数据、安全防护 | 技术架构师 |
| 30-40分钟 | 客户方实操体验 | 让客户亲自操作（关键！） | 售前工程师 |
| 40-45分钟 | Q&A + 下一步 | 收集反馈，锁定下一步 | 销售负责人 |

### 4.2 分角色演示重点

**决策层**（局长/处长）— 15分钟
- 重点：业务价值、政策合规、可量化成果
- 数据：用真实业务数据，"原来3天 → 现在2小时"

**业务层**（科室负责人）— 30分钟
- 重点：操作便捷性、"减轻工作量"的具体体现
- 演示：完整走通1-2个高频事项

**技术层**（信息中心）— 60分钟
- 重点：架构设计、接口规范、运维便利性
- 演示：接口联调实操、性能压测数据

## 五、POC 项目管理

### 5.1 组织架构

| 角色 | 职责 | 人员要求 |
|------|------|---------|
| POC项目经理 | 整体协调、客户沟通 | 有政府项目经验 |
| 售前工程师 | 环境搭建、演示脚本、现场演示 | 对产品非常熟悉 |
| 技术架构师 | 架构设计、技术难点攻关 | 深度技术专家 |
| 销售负责人 | 商务关系维护 | 参与过同类项目投标 |

### 5.2 进度计划

| 阶段 | 时间 | 交付物 |
|------|------|--------|
| 需求确认 | Day 1-3 | POC范围确认表（客户签字） |
| 环境搭建 | Day 4-7 | POC环境就绪 |
| 功能配置 | Day 8-12 | 功能就绪，内部测试通过 |
| 演示彩排 | Day 13 | 内部彩排，问题修复 |
| 正式演示 | Day 14-15 | 客户现场/远程演示 |
| 验收总结 | Day 16 | POC验收报告，客户签字 |

## 六、POC 后续动作（关键！）

> ⚠️ POC结束 ≠ 工作结束

### 必做动作
1. **3天内**提交POC总结报告（含验收报告、性能数据、客户反馈）
2. **1周内**跟进客户决策进展
3. 将POC成果转化为投标素材（技术方案中引用POC数据）
4. 维护POC环境可用案例（后续其他客户可复用）

### POC成果在投标中的使用

| POC成果 | 投标使用方式 |
|---------|-------------|
| POC验收报告（客户签字） | 技术方案附件，证明技术可行性 |
| 性能压测数据 | 技术参数响应表的支撑材料 |
| 接口联调报告 | 证明与现有系统整合能力 |
| 客户使用反馈（书面） | 典型案例补充，真实性强 |

## 八、常见问题与应对

| 客户问题 | 应对话术 |
|---------|---------|
| "POC环境能直接用于生产吗？" | "POC主要用于功能验证，生产环境会根据等保三级要求重新规划架构。" |
| "你们和其他厂商的POC有什么区别？" | "我们的POC使用真实脱敏数据；而且开放接口文档，贵方技术团队可以自行验证。" |
| "POC要收费吗？" | "本次POC是我方诚意展示，不收取费用。我们希望通过POC让贵方真实体验系统能力。" |
| "POC验收后，下一步是什么？" | "POC验收通过后，我们会根据贵方正式需求，提供完整技术方案和报价。" |
